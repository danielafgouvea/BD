PARTE 1 — Texto 
Leia com atenção o resumo abaixo antes de responder às questões.
1. Bancos de Dados NoSQL
São bancos que não usam o modelo tradicional de tabelas (relacional).
 São ideais para grandes volumes de dados, alta velocidade e sistemas que precisam crescer de forma distribuída.

2. Cassandra
O Cassandra é um banco de dados NoSQL orientado a colunas.
 Ele é projetado para trabalhar com vários servidores ao mesmo tempo, formando um cluster.
Principais características:
Alta disponibilidade


Escalabilidade horizontal


Distribuição automática dos dados entre os nós


Tolerância a falhas



3. Replicação
A replicação copia os dados entre diferentes nós para evitar perda de informações.
 Se um nó falha, outro assume, garantindo que o sistema continue funcionando.

4. Adição de novos nós
Quando um novo nó é acrescentado no Cassandra:
Ele entra vazio


O cluster redistribui os dados de forma automática


O sistema continua funcionando durante o processo


Isso permite que o banco cresça sem parar.
